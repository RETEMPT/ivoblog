# iV0 Blog 工程改进计划

最后更新：2026-06-21

## 当前基线

- 公开博客与管理端都通过 `eslint`、`tsc --noEmit`、`next build`。
- `check-data-sync.mjs` 覆盖文章、随笔、瞬间、关于页、站点配置和上传资源。
- 管理端启动器使用动态端口选择，不再清理任意占用端口的进程。
- 本地同步和 VPS 发布前的镜像复制增加路径边界校验，禁止同步到管理端自身。

## 固定验证流程

每轮功能或清理改动后按顺序执行。Windows 下不要用根目录脚本、`cmd &&` 链或嵌套 npm 编排 `next build`；请在各自项目目录里直接运行。

```powershell
node ivoblog\scripts\check-data-sync.mjs

cd E:\iV0Blogs-main\ivoblog\blog
npm run build
npm run lint
npm run typecheck

cd E:\iV0Blogs-main\ivoblog\my-blog-manager
npm run build
npm run lint
npm run typecheck
```

验证内容：

1. 管理端与公开博客的数据/资源镜像一致。
2. 公开博客 `build`、`lint`、`typecheck`。
3. 管理端 `build`、`lint`、`typecheck`。

## 迭代顺序

### P0：安全启动与同步可靠性

状态：已完成首轮。

- 恢复 Next.js 构建时 TypeScript 校验。
- 避免管理端误杀同端口的非项目进程。
- 阻止重复管理端实例并发写同一套本地内容。
- 同步操作只允许写入合法目标 Blog 目录。
- 镜像复制时清理旧文章、旧随笔、旧瞬间和旧上传资源。

### P1：后端 API 防护与错误可观测

- 给发布、同步、音乐、草稿等写操作补更明确的输入校验。
- 把常见失败原因写入统一日志，前端只显示安全摘要。
- 对远程发布命令继续收紧参数白名单，避免配置误填造成危险命令。
- 为关键 API 增加轻量级单元测试或脚本级回归检查。

### P2：内容源治理

- 明确 `my-blog-manager` 为内容源，`blog` 为发布镜像。
- 将可镜像清单集中维护，避免新内容类型只同步一半。
- 增加孤儿资源扫描，列出未被文章、配置、歌单引用的上传文件。
- 对配置中的敏感字段继续从发布镜像中过滤。

### P3：前端稳定性与性能

- 设置中心信息架构首轮已完成：按基础、视觉、内容、集成、发布分组，站点资料面板改为预览加分区表单。
- 保持 `lint/typecheck/build` 作为硬门禁。
- 优先优化首屏重资源、图片尺寸、播放器资源加载和 3D/动效降级。
- 给核心页面补最小可用的浏览器冒烟检查：首页、文章页、音乐页、设置页。
- 清理未使用依赖与重复组件，避免视觉升级继续堆叠旧实现。

### P4：部署流程收口

- 将 Vercel、GitHub Pages、VPS 三条发布路径拆成清晰的前置检查。
- 发布前自动运行数据同步检查和构建检查。
- 对远程路径、分支名、仓库 URL、SSH key 使用场景输出更明确的失败提示。
- 保持 `.env.local`、部署密钥、运行态文件不进入 Git。

## 清理原则

- 不直接删除未确认来源的用户内容；需要镜像清理时，以管理端内容为准。
- 不为了视觉效果牺牲构建、类型检查和同步一致性。
- 不引入新的大型依赖，除非能替代明显复杂且高风险的自研逻辑。
- 每次改动都要能被 `verify-project` 复核。
